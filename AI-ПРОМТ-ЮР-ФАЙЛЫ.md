# ПРОМТ ДЛЯ ИИ — адаптация юридических HTML-файлов под текущий сайт

## КОГДА ЧИТАТЬ
Когда пользователь говорит: «подставь данные», «отредактируй юридические файлы», «сделай как в прошлый раз», «почитай промт» — сразу прочитай и следуй этому документу.

---

## КЛЮЧЕВЫЕ ПРАВИЛА (НАРУШАТЬ НЕЛЬЗЯ)

1. **Менять ТОЛЬКО текст/данные.** Стили, цвета, отступы, Tailwind-классы, html-структуру — НЕ ТРОГАТЬ.
2. **Правильные данные — только из index.html и из блока реквизитов внизу.** Ничего не выдумывать, не додумывать.
3. **Правильные реквизиты (везде одинаково):**
   - ИП Павлов Михаил Николаевич
   - ИНН 462013005261
   - ОГРНИП 513737505219841
4. **Название сайта** — взять из `<title>` или логотипа в index.html
5. **Город** — взять из index.html
6. **Адрес/телефон** — если в index.html плейсхолдеры (типа `[Ваш Адрес]`, `+7 (___) ___ __ __`) — ставить их же, ничего не придумывать
7. **Email'ы — ВСЕГДА УДАЛЯТЬ.** Никаких email'ов в юридических файлах быть не должно. Если пользователь лично дал конкретный email — тогда указать его. Иначе — удалить полностью, без замены на плейсхолдер.
8. **Папку Doxs НЕ ТРОГАТЬ.** Там делать нечего. Она перегенерируется командой `npm run build`. Правки вносятся ТОЛЬКО в файлы в `src/`.
9. **ВСЁ скопированное с интернета (чужие домены, имена, ИНН, ОГРНИП, адреса, телефоны, email'ы) — удалять без остатка.** Заменять только на актуальные данные текущего проекта из index.html.
10. **«Интернет-магазин», «интернет-магазина» и любые их формы — ВСЕГДА заменять на «сайт»/«сайта»** соответственно.

## 3 ФАЙЛА ДЛЯ ПРАВКИ (все в src/)

### 1. privacy.html
- Заменить чужой домен/название сайта → название из index.html
- Убрать «интернет-магазин», «интернет-магазина» (заменить на «сайт»/«сайта»)
- Заменить чужое название (если есть в title) → название из index.html
- Блок реквизитов внизу обычно уже правильный (Павлов М.Н.) — проверить
- Все email'ы — удалить

### 2. consent.html
- Заменить чужое имя/ИНН оператора → `Павлов Михаил Николаевич ИНН 462013005261`
- Заменить чужой домен → название из index.html
- Все email'ы — удалить
- Telegram/ссылки на ботов — удалить (оставить только слово «Telegram» если нужно по смыслу, либо убрать совсем)
- Блок реквизитов внизу проверить

### 3. disclaimer.html
- Полностью переписать текст (обычно там чужой контент — недвижимость, стройка, медицина и т.п.)
- Новый текст должен соответствовать тематике текущего проекта (см. index.html):
  - Не является публичной офертой
  - Результаты индивидуальны
  - Точные условия — после индивидуальной консультации
  - Материалы для 18+
  - Изображения иллюстративные
  - Вопросы — по контактам на сайте
- Блок реквизитов внизу проверить
- Все email'ы — удалить

## ЧТО ДЕЛАТЬ ПОСЛЕ ПРАВОК

1. Проверить grep-ом, что НЕ ОСТАЛОСЬ старых данных (чужие имена, домены, ИНН, email'ы)
2. Проверить grep-ом, что НЕ ОСТАЛОСЬ ни одного email'а (если пользователь не дал конкретный)
3. Если пользователь скажет — запустить `npm run build`
